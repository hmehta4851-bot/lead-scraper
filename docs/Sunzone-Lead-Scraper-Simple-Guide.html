1. What This System Does

The system searches the internet for businesses that may buy Sunzone products. It collects available company and contact details, removes known competitors and duplicates, groups the lead under the correct Sunzone vertical, and saves it in the existing Google Sheet.

3. Keyword System

The master library contains 29,373 keywords. Every keyword is unique after ignoring capital letters. Each product keeps its own position in the library, so the system moves to a new keyword on the next run instead of repeating the same one forever.

  • All 18 product groups from the workbook are included.
  • Words containing [city] are automatically changed to the current city.
  • If the library file is temporarily unavailable, safe backup keywords are used.
  • One keyword per product is used in each run to keep runtime controlled.
The full 29,373-keyword library is preserved, but it is intentionally not searched in one day. Doing that would require hundreds of thousands of website visits and would cause blocking and very long runs. The system rotates through it gradually.

5. How Bad Leads Are Reduced

  • Own-brand block: Sunzone names, website and email domain are rejected.
  • Competitor block: known competitor names and domains are rejected.
  • Phone check: only leads with a usable Indian phone number reach the sheet.
  • Global duplicate check: the phone is checked across every live vertical tab.
  • Source balance: results are selected in rounds so one large source cannot occupy every slot.
  • Lead score: phone, email, contact name, designation and website improve the score.
No public-web scraper can promise that every business is a genuine buyer or that every competitor in India is known forever. The system blocks the configured competitors and records the search source, but sales verification remains necessary.

6. What Appears in Google Sheets

The live tabs are: Playful Leads, Graceful Leads, Powerful Leads, Joyful Leads, Acryplay Leads, Track & Field Leads, Sports Equipment Leads and Woodplay Leads.

Each saved row contains:

Date, City, Vertical, Product, Search Keyword, Company Name, Contact Person, Phone, Email, Designation, Source, Website and Lead Score.

Before the new structure was started, old product tabs were copied into timestamped archive tabs and then cleared. This preserves recoverability.

7. What Runs Automatically

  1. The main scraper is scheduled for 8:00 AM India time, Monday to Saturday.
  2. It chooses the next city and a new keyword for each product.
  3. It attempts all nine sources for each selected keyword.
  4. It writes only new phone numbers to the Google Sheet.
  5. It emails start, progress and completion information.
  6. A watchdog checks at 11:00 AM India time.
  7. If the main run did not complete, the watchdog automatically triggers another run.
Normal daily operation requires no action from the owner. Your practical job is to review the Google Sheet and pass qualified leads to salespeople.

8. Recovery and Safety

  • Each website source is isolated, so one failure does not crash the product.
  • Browser sources use separate sessions, preventing one website from contaminating another.
  • Progress is saved atomically, reducing the chance of a damaged state file.
  • Two scraper runs cannot overwrite each other because GitHub concurrency is enabled.
  • If existing phones cannot be loaded from the sheet, the run stops and retries instead of risking duplicates.
  • Old data is archived before a reset.

10. Billing in Simple Terms

The daily scraper does not call Claude, Codex, Gemini, OpenAI or Anthropic APIs. Codex and Claude Code were used to build and review the code, but they are not required for the daily run.

The automation runs on GitHub Actions and writes through Google Sheets authentication already configured in the repository. GitHub and Google usage remain subject to the limits of the accounts attached to them. The code itself does not add a paid AI API bill.

11. What You Should Watch

  • Completion email: total new leads and source health.
  • Google Sheet: whether phone, company and product match the intended lead.
  • Repeated zero-yield sources: websites can change their layout or blocking rules.
  • Competitor list: add a newly discovered competitor when sales identifies one.
Bottom line: the system is designed to be autonomous, transparent and recoverable. It can significantly reduce manual prospecting, but it should support sales judgment rather than replace it.
